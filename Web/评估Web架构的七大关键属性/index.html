评估Web架构的七大关键属性

HTTP 协议应当在一下属性中取得可接受的均衡：

  1. *性能 *Performance **：影响高可用的关键因素
  2. 可伸缩性 Scalability：支持部署可以互相交互的大量组件
  3. 简单性 Simplicity：易理解、易实现、易验证
  4. 可见性 Visible：对两个组件间的交互进行简书或者仲裁的能力。如缓存，分层设计等
  5. 可移植性 Portability：在不同的环境下运行的能力
  6. 可靠性 Reliability：出现部分故障时，对整体影响的程度
  7. 可修改性 Modifiability：对系统做出修改的难易程度，由可进化性，可定制性，可扩展性，可配置性，可重用性构成

性能

  • 网络性能 Network Performance
    • Throughput 吞吐量：小于等于带宽 bandwidth
    • Overhead 开销：首次开销
  • 用户感知到的性能 User-perceived Performance
    • Latency 延迟：发起请求到响应的时间
      • 图片开始加载，每接收到一部分数据就渲染一一部分
    • Completion 完成时间：完成一个应用动作所花费的时间
      • 图片加载完成
  • 网络效率 Network Efficiency
    • 重用缓存，减少交互次数，数据传输距离更近 ( CDN )，COD

可修改性

  • 可进化型 Evolvability：一个组件独立升级而不影响其他组件
    • 浏览器升级，服务端不变，正常使用
  • 可扩展性 Extensibility：向系统添加功能，而不会影响到系统的其他组件
    • SSL 加密，不使用 SSL 加密的网站正常使用
  • 可定制性 Customizability：临时性、定制性地更改某一要素来提供服务，不对常规客户产生影响
  • 可配置性 Configurability：应用部署后可通过修改配置提供新的功能
    • nginx*， *apache
  • 可重用性 Reusability：组件可以不做修改在其他应用使用
    • WebKit 渲染引擎
